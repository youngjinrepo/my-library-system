## 도서정보 (BookInfo)
_Entity_
#### 속성

- 서명 (title) : 도서의 제목
- 권호 (volume) : 도서의 권, 호
- 저자 (author) : 도서의 저자
- ISBN (ISBN) : 국제표준도서번호
- marc (marc) : 기계가 읽을 수 있는 목록
- 분류 (classify) : 도서의 주제 
- 성인 도서 (is adult) : 성인 도서 여부
- 발행자 (publisher) : 도서의 발행자

#### 행위

- 수서 (acquisition) : 자료를 선별하여 구입, 기증
- 편목 (cataloging) : 도서를 도서관에 맞춰 데이터를 생성, 전산에 입력하는 과정
- 제거 (remove) : 책을 도서관에서 제거한다.

#### 규칙
- 배가 되어있어야 조회가 가능하다.
---
## 도서 정보 상태 (BookInfoStatus)
_Enum_
#### 속성
- DRAFT : 책 정보가 입력된 후 대기 중인 상태
- CATALOGING : 편목,책에 기본 정보를 입력하는 상태
- COMPLETED : 편목 작업까지 완료 된 상태
- UNAVAILABLE : 어떠한 이유로 책을 이용할 수 없는 상태
---
## 도서 (BookItem)
_Entity_
#### 속성
- 등록번호 (code) : 도서관내 유일한 도서의 코드
- 청구기호 (call no) : 도서관내 책의 위치를 나타내는 기호
- 대출 상태 (status) : 도서의 상태 (대출 가능, 대출 불가, 예약 불가)
- 구역 (location) : 도서관내 도서가 위치한 구역
#### 행위
- 배가 (shelving) : 서가에 도서를 꽂는 행위로 청구기호를 발행하는 것을 배가로 정의
- 제거 (remove) : 분실, 훼손 등 물리적 책이 없어지는 걸 의미
#### 규칙
- 배가 되기 전엔 청구기호와 등록 번호가 부여 되어 있어야 한다.
- 등록 번호는 도서관내 유일한 코드
- 등록 번호는 도서관코드 + 번호로 구성된다.
---
## 도서 상태 (BookItemStatus)
_Enum_
#### 속성
- CATALOGING : 편목, 도서 정보가 생기고 편목 진행 중인 상태
- SHELVING : 편목이 완료되어 서가에 도서가 있는 상태
- LOANED : 이용자가 도서를 대출한 상태
- UNAVAILABLE : 어떠한 이유로 책을 이용할 수 없는 상태

---
## 이용자 (Member)
_Entity_

도서관을 이용하는 이용자
#### 속성
- id (id) : 회원의 고유 식별자
- 이름 (name) : 회원의 실명
- 이메일 (email) : 도서관 웹 이용을 위한 아이디, 이메일 번호와 겸용
- 성별 (sex) : 회원의 성별 (남,여,선택 안함)
- 나이 (age) : 회원의 나이
- 주소 (address) : 회원의 집주소
- 가입일 (join date) : 회원이 도서관에 id를 생성한 날
- CI (ci) : 실명 인증 후 유저의 해시 값
- 회원등급 (grade) : 회원의 등급
- 상태 (status) : 회원의 상태 (정상, 제적, 휴먼, 탈퇴)
#### 행위
- 대출
- 반납
- 반납 연기
- 가입
- 등록
- 예약
#### 규칙
- 개인 정보 중 민감 정보는 제공 여부를 선택하여 수집하지 않을 수 있다.
- 반납 연기는 예약자가 없을 때 만 가능핟.
- 실명 인증이 되어야 정회원이 되어 대출, 예약 등 도서관 서비스를 이용할 수 있다.
---
## 대출 (Loan)
_Entity_
#### 속성
- 대출키 (id) : 대출 이력의 고유 식별자
- 회원 번호 (member no) : 대출한 회원의 식별자
- 등록 번호 (reg no) : 대출된 도서의 식별자
- 대출일 (loan date) : 대출일
- 반납 예정일 (due date) : 도서의 반납 예정일
- 반납 연기 횟수 (renewal cnt) : 반납 연기 횟수
- 반납일 (return date) : 도서의 반납 일
#### 규칙
- 반납 예정일 산정 시 도서관의 휴관일이 있을 경우 해당 일수 만큼 +1 한다.

## 사용자 (User)
_Entity_
도서관 관리 시스템의 사용자
#### 속성

## 도서관 (Library)
_Entity_

#### 속성
- 이름 (name) : 도서관의 이름
- 휴일 (holiday) : 도서관의 휴관일
- 코드 (code) : 도서관의 부호, 시스템 상 다른 도서관과 구분하기 위해 사용 ㄴ됨
#### 규칙
- 도서관 코드는 3자리 대문자 영문

### 도서관리정책 (policy) Value Object
#### 속성
- 대출 기한
- 대출 가능 건수
- 예약 가능 건수
- 한줄 서기 사용 여부
- 반납 연기 가능 횟수
