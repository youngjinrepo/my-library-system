ADR (Architectural Desision Records) : 의사 결정 기록

#### 대출 정책 분리
- 상황 : Libaray 엔티티의 대출 기간, 최대 대출 건수에 앞으로 추가될 속성들 까지 너무 많은 값들이 나열 될 것으로 예상되어 조치가 필요하다고 판단
- 결정 : 우선 Value Object로 묶어 대출 정책으로 분리, 추후에 설정 값들이 추가되면 그때 정책명을 다시 분리하기로 결정
---
#### 도서(Book) 도메인 분리
- 상황 : 기존 도서의 Entity는 동일한 책이 여려권 있을 경우 그 정보를 그대로 담기가 어려움
- 결정 : Book을 공통 정보를 담은 책 정조 BookInfo와 실제 책의 물리적 정보 BookItme으로 분리
---
#### 이용자 남은 대출일 수 계산 방식
- 상황 : 반납 예정일을 매번 계산할 경우 다수 이용자 조회 시 많은 시간 소요
- 결정 : 이용자의 남은 대출일수(or연체일수)는 매일 자정에 계산
---
#### Exception 상세 관리
- 상황 : 예외를 식별해내기 어려움, 예외 메시지를 보고 식별 해야 함
- 결정 : 비즈니스 상황에 맞춘 Exception 분류 적용
---
#### 상위 Policy 추상화 제거
- 상황 : 정책이 늘어나며 Policy에 다 관리할 경우, Service가 비대해지고 각 Entity에서 받자니 의존성이 높아짐
- 결정 : 각 domain 별 policy로 관리
